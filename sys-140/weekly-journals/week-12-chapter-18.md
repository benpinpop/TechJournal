### Security Policy

A security policy is a global policy ensuring the safe use of devices. It must be written down for it to be as effective as possible.

There are physical access policies, determining who can enter a building, what they need to do it, etc. An acceptable use policy defines how devices should be used and who should have access to devices.

You can also have password policies (length and complexity, how often it should be changed), email usage policies (when an email is deleted, how often it's backed up, etc), policies to comply with regulations, policies that define who gets remote access, and emergy procedure policies.

Security policies are important as they serve to protect the company against breaches.

### Physical Security

Physical Security encompasses everything physical, like building security. Keys, guards, bollards, locks, and access control systems are all examples of physical security.&#x20;

Electronic access control systems are a common security control that allows employees to use electronic key cards. They can be easy to program and revoke; everything is stored in a database and logged easily, you can add layers of control, and you can easily disable a card if it's lost.&#x20;

There are a variety of physical security devices. Here are some highlights:

* Smart Card - Like a government ID
* Key fob - Keyless entry (Car key)
* Radio frequency ID (RFID) - Key card like the IDs we use to get into the dorm
* Badge reader - Reads ID cards
* Guard - Self-explanatory
* Security Token - A physical authentication token that provides access to a system.
* TPM - Module on a motherboard that holds credentials, handles authentication for a computer
* Locks - Self-explanatory

Physical Security can also be done with biometrics, by verifying a biological factor like a fingerprint, voice sample, retina scan, or facial recognition. Biometrics are much stronger than other forms of security as they cannot be easily bypassed. The downside of biometrics is that they are currently more expensive to implement.&#x20;

Physical Security also includes document security. Documents might require shredding or burning, which can be done by vendors or by physical shredder devices. Shredding stops individuals from dumpster diving and accessing confidential or classified information.&#x20;

### Logical Security (skip the 'BIOS/UEFI' subsection)

Logical Security is security through software. There are numerous ways of protecting a computer through logical security, like anti-malware, firewalls, directory permissions (restricting a user from accessing another user's folder), VPNs to hide traffic, and disabling unused ports and filtering emails. You can also implement MAC address restrictions on certain ports so you only allow certain devices to connect to a physical port. All of these security processes protect individuals from infection and harm.

#### Authentication

Authentication refers to verifying the identity of someone before allowing them to access a service. Most commonly, we use 2FA (2-factor authentication) or even multifactor authentication. You might need a smart card or to verify by text message to get in. This provides a lot more security because it requires an attacker to have much more information to conduct a successful attack.&#x20;

A soft token is a way to authenticate through a changing token generated by an authenticator app, functioning as a one-time password for login.&#x20;

There are also Remote Authentication Dial-in User Service (RADIUS) and Terminal Access Controller Access-Control System (TACACS+), which centralize authentication, authorization, and accounting (AAA). Another protocol is Kerberos, an alternative to SSH, POP, or SMTP, stopping confidential packets from being changed or read.&#x20;

#### Login OS Security Options

To get into any device, you typically need a login. There are multiple types of ways to authenticate on a device, from a username and password, a pin (personal identification number), fingerprint, facial recognition, and single sign-on, which allows you to sign onto multiple devices with your username and password.&#x20;

Advisory recommendations for login security are to never write down passwords, use complex passwords, do not use consecutive letters or words, and make passwords expire after a certain amount of time. Implement a minimum and maximum password age, don't allow individuals to reuse passwords, log off with Windows Key+L, set a failed password attempts limit, and get a privacy filter so people can't shoulder surf.&#x20;

To solve the problem of weaker passwords, implement the use of a password manager that can centrally store your passwords.

### End-User Education

Educating the individual about cybersecurity is the best way to prevent incidents, as humans are the weakest link. For example, security awareness training, or presenting the acceptable use policy every time a user logs in, is a good way to ensure an individual is aware of an issue.&#x20;

Common symptoms of a virus or malware are:

* Missing files or renamed files
* Weird directories appearing
* Anti-virus detecting malware
* Applications not loading
* Unusual messages on the screen
* Computer not booting or applications not loading
* Reduced hard drive space or increasing amounts of bad sectors on hard drives.

Anti-malware scans should be run regularly, and all files should be scanned upon download. Anti-malware is typically capable of quarantining files that it considers dangerous.&#x20;

#### Social Engineering

Social Engineering is a way of gaining information, typically via impersonation, whether it be by email, phone, or another online medium. Shoulder surfing (looking over your shoulder to get passwords) and dumpster diving (looking in the trash to get info) are also social engineering (I did not know this...). Tailgaiting is walking behind someone to gain access, so if the person in front of you has a key card, they let you in because you're right behind them.&#x20;

#### Phishing

Phishing is a type of social engineering involving the extraction of information via an email message that appears legitimate.&#x20;

There are three types of phishing: spear phishing, whaling, and vishing.

Spear phishing is a targeted form of phishing where an attacker has detailed information about you, and so they tailor a message to be more effective at extracting information.

Whaling targets high-level and senior executives, knowing that they have higher-level access than most.&#x20;

Vishing is phishing via a phone call, rather than email.&#x20;

There are many ways to recognize a phishing email:

* Check the sender. Do you know who it is? Is it a spoofed email address?
* Check for any spellings or typos
* Does it come from the IT department? Is it asking you to do a certain thing?
* Hover over any link to see the address before you click it. Does it look suspicious?

Scareware is a message scaring you and giving you a link to fake antimalware application that could cause harm to your system.

#### Security Attacks

Security Attacks can come from insider or outsider threats. Backdoors provide a way in from inside the network. Brute force and dictionary attacks are password attacks to try and gain access to accounts. Denial and Distributed Denial of Service (DoS & DDoS) can overload systems. An evil twin is a WiFi attack meant to capture information and packets from users. A man-in-the-middle attack (on-path) captures network information between a sender and receiver. Password spraying uses commonly used passwords. Rainbow tables are a method to crack common hashes. Replay attacks involve recreating a packet from an individual and resending the packet. (These can be prevented with time stamps and nonces). Smurfing uses ICMP (DoS?) to overwhelm a network by sending too many requests.

Spoofing sends a fake packet/frame with a fake MAC address. (Google this one) SQL injection allows an individual to access a website's database via a cross-site scripting attack.&#x20;

### Permissions

Permissions are a way of designating what can or cannot be done, like viewing files or connecting to devices. Without permissions, anyone would be able to access anything. There are two types of permissions in Windows: share permissions and NTFS permissions.&#x20;

Share permissions give or limit access to data on the network.&#x20;

NTFS permissions are stronger than share permissions, and can only be used on NTFS drives.

#### Share Permissions

To share a folder, go to Properties > Sharing and add a user. This will allow other users on a computer to access the file, even if they are not logged in as you. You can also set permissions in Advanced Sharing, like Change and Read. There is also full control. Change allows you to add files, subfolders, and change the file entirely. Full Control allows Change and Read.

**All folders inherit the permissions of their parent folder, or the folder that contains the subfolder.**

A local share is a shared folder, media device, or printer shared over the network. Administrative shares are different from local shares, primarily containing Windows files, starting with a $ sign. These are automatically created by the system. A hidden share can also be made by adding the $ sign to it.

You can also share files using the "Public" folder, which allows anyone with a user account to access the data (C:\Users\Public).

#### NTFS Permissions

On an NTFS partition (a type of hard drive partition), you have access to more security controls, including: Full Control, Modify (list items, read and write, but cannot delete or take ownership), Read & Execute (cannot delete or change, can execute applications), List Folder Contents (look inside, but not read), Read (Display folder attributes and permissions, and read files) and Write (Can add folders, change and add files).&#x20;

Each subfolder is given permissions from the parent object. Copying a file inherits the destination folder permissions. Moving a file folder keeps the same permissions. Copying a folder to a different file system (FAT) results in the loss of all permission setup.

Deny overrides allows, meaning even if a user is allowed to read a folder, if they have a deny permission set anywhere, it overrides this. Folder permissions also add up, so user and group permissions will be cumulative. You can see this in the effective access tab.&#x20;

### Protecting The Operating System And Data

#### Introduction section&#x20;

There are a variety of ways to protect your Operating Systems and your data:

* Use NTFS.
* Update your computer often. (patch management)
* Update your OS if it has reached end-of-life.
* Have an alternative boot source in case the current OS boot fails.
* Encrypt your data.
* Place OS files and data on separate partitions.
* Update your firmware and drivers often.
* Share only the folders you need to
* Enable a password on the BIOS and disable ports to avoid external devices infecting the device.&#x20;

#### Backup/Restore

Backups are an extremely useful way of protecting your data, ensuring that you have a way to restore your information even if your drive corrupts itself. While hardware can be replaced, data cannot. Backups should be performed frequently.

There are multiple types of backup media, but backups were originally stored on magnetic tape, although we now use optical discs, USB drives, external drives, and most importantly, **cloud storage.**

Local storage is cheap and is easily secured, but you have to maintain and store it. Cloud storage has security concerns, as it can be accessed online, and you also need an internet connection to restore anything.

There are different types of backups.

**Full Backups** - Backs up all files

**Incremental Backups** - All files that have changed

**Differential Backup** - All files that have changed since the last full backup.

**Synthetic** - A mix between a fullbackup and an incremental backup.

Many pieces of data cannot be recovered once they are lost, despite the availability of third-party recovery software.&#x20;

#### Account Recovery Options

If you end up losing or corrupting data, you might need to recover your accounts, so it's important to document critical information about your device beforehand, like computer model and warranty, IP addresses, firewall rules, account names and passwords, BIOS passwords, and software keys.&#x20;
