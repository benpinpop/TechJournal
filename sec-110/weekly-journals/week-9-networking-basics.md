### Vocabulary & Key Terms

#### Network

* A group of interconnected devices that communicate and exchange data

#### Network Topology

* How a network is arranged
  * Bus - Single Cable
    * May have lots of performance issues, AVOID
  * Ring - Loop, one-directional
    * Simple, not scalable, can have performance issues
  * Star - Central Hub
    * Scalable, simple, single point of failure
  * Mesh - All devices are connected to each other, creating a mesh
    * Expensive, very reliable

#### Server

* Provides access to data, resources, or programs.

#### Hub

* Sends incoming data to all network devices. A very simple network device.

#### Switches

* Connects devices and directs packets, and can scan packets. A switch is smarter than a hub.

#### Routers

* Connects networks and directs packets.

#### Bridges

* Connects network segments to act as a single network

#### Access Points

* Allows a wireless device to access a wired network.

#### Network Types

* LAN - Local Area Network (Business, or Home)
* WLAN - Wireless Local Area Network (Business, or Home)
* WAN - Wide Area Network (School, or Large Business)
* MAN - Metropolitan Area Network (City)
* PAN - Personal Area Network (Personal, Bluetooth)

#### Packet

* A piece of information is sent across the network with information on the recipient and sender, almost like a letter. Contains headers.

#### IP Address

* A unique identifier assigned to a device that changes based on the network connected. An IPv4 address consists of a network ID and a host address (e.g. 192.168.1.1). IP addresses are made up of 4 octets, being 32 bits.

#### MAC Address

* A unique identifier assigned to a device that does not change. It is placed somewhere on the hardware of the device and is unique to all computers.

#### Subnet Mask

* Defines how many bits are in the Net ID.
  * For example, a subnet mask of 255.255.255.0 or /24 indicates the first 24 bits are part of the network ID.&#x20;

#### Domain Name System

* A server (with a protocol) that translates website names into IP addresses.

#### OSI Model

* A standardized model for describing how networks function. Split into 7 layers.

#### TCP/IP Model

* Another way of describing how networks transfer and process data. Split into 3 layers.

### Main Ideas and Takeaways

* The internet and its networks are highly complex, using a variety of protocols to be able to function. These protocols are extremely robust, making sure that using networks is reliable and easy for all users.&#x20;
